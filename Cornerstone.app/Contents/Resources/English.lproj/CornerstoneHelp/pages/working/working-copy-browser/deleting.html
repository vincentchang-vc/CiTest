Deleting Working Copy Items

Delete items in a working copy by performing the following steps:

  1. Select the items to be deleted.
  2. Remove the items by selecting File > Delete (key equivalent ⌘⌫), clicking the Delete toolbar button or using the Delete function on the item's right-click menu.

    If the selected item has been been modified since the last update or commit, Cornerstone prompts to copy the item to the Trash before performing the delete. Selecting Delete Anyway will result in modifications being irretrievably lost.

  3. The items are scheduled for deletion.

Files are deleted from the file system immediately. When deleting folders, all files are removed from the folder and all sub-folders, but the skeleton folder structure remains in place until the deletion is committed to the repository.

The skeleton folders will be removed after the deletion has been committed to the repository.

Do not delete the skeleton folders manually before you have committed the changes.

Undoing a Delete

You can use the Revert function to undo a delete.

Note the items will be reverted back to their BASE revisions. Any modifications made prior to the delete will not be restored.
